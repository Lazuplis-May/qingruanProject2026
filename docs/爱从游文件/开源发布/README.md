**1. 任务背景**

为培养同学们的开源协作开发能力，本次实训课程要求所有开发小组按照开源软件开发规范进行代码管理与发布。请各小组认真阅读以下要求并按时完成。



**2. 任务要求**

​	**1. 开源平台注册与账号绑定**

- 所有小组成员必须在全国示范性软件学院联盟推荐的开源生态服务平台（太乙平台）完成注册
- 注册地址：https://www.taiyi.top/home
- 注册后，请按照《太乙开源生态服务平台使用指南》绑定自己的Gitee或GitHub账号
- **重要**：组内所有成员必须使用相同的代码托管平台（统一使用Gitee或统一使用GitHub）



​	**2. 代码仓库创建与协作开发**

- **使用在太乙平台绑定的Gitee/GitHub账号**创建小组项目代码仓库
- 所有组员都必须参与代码提交，在项目开发周期内应有合理的commit记录
- 代码提交应符合开源项目规范，包括有意义的commit message、合理的目录结构等

​	**3. 最终提交内容**

- 源代码打包文件（如zip或tar.gz格式）
- 完整的项目文档说明，包括：项目简介、功能说明、安装部署指南、使用说明、开发成员及分工
- **必须**在文档中附带开源仓库的完整地址

**3. 注意事项**

\1. **开源仓库建设**：(1) 确保仓库结构清晰，代码组织合理，避免杂乱无章的文件堆积; (2) 每位成员都应参与代码提交，并在commit message中清晰描述修改内容; (3) 项目文档应完整，包括必要的README、开发指南和使用说明。



\2. **代码质量要求**：(1) 代码应符合编程规范，具有良好的可读性和可维护性；(2) 功能实现应完整，避免出现未完成的模块或严重Bug；(3) 关键代码应有适当注释，便于评审和后续维护



\3. **开源仓库提交要求**：(1) 提供的仓库地址必须有效，评审时将直接访问该仓库进行代码检查；(2) 确保所有组员的commit记录可见，体现团队协作过程；(3) 最终提交的打包代码必须与开源仓库内容一致，避免版本差异。



\4. **时间安排建议**：(1) 尽早完成太乙平台注册和账号绑定；(2) 建议采用**持续集成**的开发模式，定期提交代码，而非最后集中提交；(3) 预留足够时间进行代码审查和文档完善。

如有任何问题，请及时联系课程助教或任课教师。

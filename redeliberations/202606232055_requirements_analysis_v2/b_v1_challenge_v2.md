**[问题-中度] 问题 5 的次级证据存在事实错误——"iframe 间 Token 过期同步"文字不在活动内容中**

诊断报告问题 5 声称：第 4.10 节"Token 过期处理"段落"第 347 行附近"存在"iframe 间 Token 过期同步"这一在纯 Vue3 SPA 中不存在的场景。

通过全文搜索确认，"iframe 间 Token 过期同步"这串文字唯一出现在**第 1558 行**（第 16.8 节 v6→v7 修订历史记录中描述该轮次修订操作时），而非诊断报告所声称的活动内容第 347 行。

这意味着问题 5 的次级证据定位错误：诊断报告声称活动内容（第 4.10 节 Token 过期处理）中存在与 Vue3 SPA 矛盾的 iframe 描述，但经核实该段落已正确使用 Pinia 响应式术语。实际矛盾仅存在于修订历史记录（第 1558 行），属于与问题 2/6 同类的"修订历史记录清理遗漏"问题。

**此证据错误的具体影响**：如果产出作者按问题 5 的建议去第 347 行查找并修改"iframe 间 Token 过期同步"文字，会发现目标文字根本不存在于活动内容中，导致该条修订建议无法执行。

- **问题**：诊断报告问题 5 声称第 4.10 节"Token 过期处理"段落"第 347 行附近"存在"iframe 间 Token 过期同步"这一与 Vue3 SPA 矛盾的文字。经逐行核实，第 347 行的实际文本为"**Token 过期全局同步**"，内容正确使用了 Pinia 响应式术语（`authStore.clearAuth()`、Pinia 响应式系统），不存在 iframe 相关描述。全文搜索确认"iframe 间 Token 过期同步"仅出现在第 1558 行（第 16.8 节 v6→v7 修订历史记录中），属于修订历史记录的描述文本，而非第 4.10 节的活动内容。

- **建议方向**：1. 将问题 5 的次级证据描述修正为：第 4.10 节"Token 过期处理"段落的活动内容已正确使用 Pinia 术语，"iframe 间 Token 过期同步"仅在修订历史记录（第 1558 行）中存在，属于修订历史记录清理遗漏（与问题 2、6 同类）。2. 将问题 5 的改进建议聚焦于其核心证据（第 310 行交叉引用），次级证据的处置合并到问题 2 或问题 6 的修订历史记录清理范畴。

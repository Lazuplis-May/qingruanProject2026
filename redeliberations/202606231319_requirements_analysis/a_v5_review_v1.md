### 2. 清晰性

**[通过]** 各功能模块的描述层次清晰，从功能概述到详细交互到验收标准，形成了可被下游团队理解的结构。

**[通过]** 访问控制矩阵（4.10 节）以表格形式逐页面标注了公开访问、需登录、需 admin 角色的三级权限边界，清晰无歧义。

**[通过]** API 接口规格（第 6 节）按功能模块分组，含 HTTP 方法、端点路径、认证要求、请求参数和响应格式，接口边界明确。

**[问题-一般]** 第 5 节数据路径数量前后矛盾。第 5 节开篇声明"平台存在**两条**数据操作路径，需明确区分"（第 297 行），但随后列出了三条路径（常规 CRUD、AI 驱动 Text2SQL、AI 内容生成持久化）。同节末尾又使用"三条路径均通过 Express 服务接口完成数据库读写"（第 303 行）的正确表述。这是 v4→v5 修订中从两条路径扩展为三条后，未同步修改前导句的遗留错误。虽然第 13.1 节修订说明明确记载了此变更，且多数上下文使用"三条"，但开篇的"两条"会给快速浏览文档的读者造成误导——他们可能在读到三条路径之前就已经形成了"两条"的认知框架，后续出现"三条"时产生困惑。

**[问题-一般]** Dify Agent 通过 Text2SQL 工具回调 `/api/admin/execute` 时的用户身份传递机制不明确。文档第 4.10 节和 6.10 节定义了 `/api/admin/execute` 的分级鉴权模型：根据 JWT Token 中的 role 和 user_id 进行行级权限约束。但该端点被 Dify Agent 调用时（经 AI 助手触发的 Text2SQL 操作），实际的 HTTP 请求来自 Dify 服务器而非用户浏览器，用户浏览器的 JWT Token 无法直接传递到该回调路径中。文档将权限保障描述为"由 Express 端点内部分级鉴权逻辑和 Dify Agent 的系统提示词共同保证"，但未说明：
- Dify Agent 调用 `/api/admin/execute` 时使用什么认证凭据（是系统级 API Key 还是某种代理 Token）
- Express 端点如何从 Dify Agent 的请求中获知当前操作对应哪个终端用户的 user_id
- 若仅依赖 Dify Agent 系统提示词约束（即提示词层面限制 Agent 只生成当前用户的 SQL），则缺乏技术层面的行级权限强制执行

此问题影响第 5 节"AI 驱动的 Text2SQL 路径"的完整性和第 4.10 节管理员鉴权模型的可靠性。下游架构设计师若按当前文档设计实现，可能发现 JWT 鉴权模型无法直接适配 Dify Agent 回调场景，需重新设计认证桥接方案。

## 修改要求

- **问题**：第 5 节开篇（第 297 行）声明"平台存在两条数据操作路径"与随后实际列出的三条路径矛盾，正确的数量应为"三条"。
- **原因**：v4→v5 修订新增了第三条路径（AI 内容生成持久化路径）但未更新前导句。文档内部"两条"与"三条"并存，下游读者可能形成错误的架构认知，导致设计时遗漏第三条路径的接口和处理逻辑。
- **建议方向**：将第 297 行的"两条"修改为"三条"，使其与第 303 行"三条路径"的表述及实际列出的路径数量保持一致。

---

- **问题**：Dify Agent 通过 Text2SQL 工具调用 `/api/admin/execute` 端点时，终端用户的身份（user_id）如何从用户浏览器传递到 Express 端点的这条调用链未被定义。当前文档假设通过 JWT Token.user_id 进行行级权限约束，但 Dify Agent 的回调请求并非来自用户浏览器，JWT Token 不可用。
- **原因**：此缺口导致 `/api/admin/execute` 的分级鉴权模型在 AI 助手 Text2SQL 场景下存在实现断层——要么行级权限约束无法实际执行（安全风险），要么下游架构师在设计阶段才发现需要补充额外的认证桥接机制（返工风险）。
- **建议方向**：在第 5 节"AI 驱动的 Text2SQL 路径"或 6.10 节 `/api/admin/execute` 端点描述中补充以下任一方案：（a）明确 Dify Agent 的 Text2SQL 工具在调用 `/api/admin/execute` 时携带系统级 API Key（如 `DIFY_ASSISTANT_API_KEY` 或独立的服务间 API Key）并在请求体中附带 user_id 参数，Express 端点支持双认证模式（JWT 用于浏览器直连场景，API Key + user_id 参数用于 Dify Agent 回调场景）；（b）明确此机制留待概要设计阶段定义，但需标注为待解决的设计决策点。无论采用何种方案，需同步更新第 4.10 节管理员鉴权描述，说明 AI 助手 Text2SQL 路径的鉴权方式区别于管理员的浏览器直连路径。

# 项目需求描述 — 糖尿病预治智能助手

## 项目背景

2026年青软实训项目（东北大学软件学院）。核心项目为"糖尿病预治智能助手"——一个融合 DeepSeek 大模型和 Dify 智能体开发技术的一站式糖尿病预治管理 Web 平台。

开发技术栈：HTML + CSS + JavaScript，配合 DeepSeek API 和 Dify 工作流平台。

## 项目概述

在当今全球化的时代，糖尿病已然成为一个令人忧虑的全球性健康难题。随着生活方式的急剧转变，如高糖高脂饮食的普遍化、运动量的大幅减少，以及人口老龄化进程的加速，糖尿病的发病率正以惊人的速度持续攀升。

"糖尿病预治智能助手"创新性地融合了 DeepSeek 和 Dify 的前沿智能体开发技术，精心构建了一个一站式的糖尿病预治管理体系。通过深度对接专业医学知识库，并运用DeepSeek大模型，能够对用户输入的健康数据进行全方位、深层次的分析，进而精准预测糖尿病风险。不仅如此，该助手还会根据用户的具体健康状况和糖尿病类型，量身定制个性化的防治方案，涵盖科学合理的饮食计划和针对性强的运动指导，助力用户有效管理病情。除此之外，还贴心地设置了与糖尿病领域专家在线交流的名医对话界面，让用户能够随时随地获得专业、权威的医疗建议。

## 产品使用范围

该平台适用于普通人与糖尿病患者，主要包括以下功能模块：首页糖尿病科普信息展示、医生在线咨询、糖尿病风险智能预测、健康生活方案生成、生活打卡、个人中心管理、打卡信息管理、健康资讯管理等。

## 非功能性需求

### 用户界面需求
- 主颜色、字体颜色、字体大小、界面布局、界面交互方式、界面功能分布、界面输入输出模式层次分明
- 简易性、记忆负担最小化、一致性、从用户观点考虑

### 产品性能需求
- 用户操作在1秒内返回结果，最多不超过3秒
- 数据同步在1s内完成
- 系统每天连续运行24小时，每周七天
- 容错性：正常操作不会导致系统出错或崩溃

### 产品质量需求
正确性、健壮性、可靠性、性能效率、易用性、清晰性、安全性、可扩展性、兼容性、可移植性

## 运行环境

- 开发工具：Visual Studio Code v1.92.2
- 开发技术：HTML、CSS、JavaScript、Dify智能体开发
- 服务器：3台，8核16G内存，Linux系统
  - 数据服务器：SQLite、Express、Nginx
  - 系统运行服务器2台：Nginx集群部署

## 并行任务轨道

除主线U+ Web开发外，实训还包含5个并行任务轨道：

### A1 - 大模型开发
- 使用 DeepSeek 和 Dify 实现 AI 能力
- 需提交文档说明如何在系统中使用大模型技术

### A2 - 鸿蒙应用开发
- 选取核心功能模块实现鸿蒙化开发
- 可以是PC端、手机端、平板端或其他嵌入式设备应用
- 需提交文档说明如何使用鸿蒙开发技术

### A3 - 国产数据库应用
- 基于金仓数据库（Kingbase）开发
- 需建3个以上数据表
- 在Web端或移动端至少完成一个与项目相关的业务功能（含增删改查）

### A4 - 单元测试
- 后端：JUnit 5，核心功能100%覆盖，辅助功能≥80%
- 前端：Vitest，核心业务逻辑100%覆盖，工具函数≥80%
- 需提交测试报告和覆盖率截图

### A5 - 性能测试
- 使用 WebRunner 工具
- 测试场景：登录接口
- 并发用户数：50、100、200
- 梯形负载模式，用户创建速率每秒5个
- 单次测试时长10分钟
- 关注指标：TPS、事务响应时间、标准差、错误率

### 开源发布
- 在太乙平台注册并绑定 Gitee/GitHub
- 所有成员参与代码提交
- 提交：源代码打包、项目文档、开源仓库地址

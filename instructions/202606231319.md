# 用户指令原文

场景：需求澄清与再审议

项目背景：
- 2026年青软实训项目（东北大学软件学院）
- 核心项目：糖尿病预治智能助手（Web应用，HTML/CSS/JS + DeepSeek + Dify）
- 并行任务轨道：A1大模型开发、A2鸿蒙应用开发、A3国产数据库（金仓）、A4单元测试（JUnit/Vitest）、A5性能测试（WebRunner）、开源发布（太乙平台）

已有材料：
1. 项目需求描述（已完成）：定义了项目概述、功能性需求（首页/医师咨询/个人中心/风险预测/生活方案/健康资讯/打卡/AI助手/管理员）、非功能性需求（UI/性能/质量）
2. 软件需求规格说明书模板（已完成框架）：含项目概述、产品范围、用户角色（普通用户/管理员）、运行环境、功能性需求分类、非功能性需求
3. 项目开发任务分解已细化到每个模块的AI工具使用、Dify工作流、DeepSeek实现

审议任务：
1. 确认需求完整性和一致性——项目需求描述 vs SRS vs 任务分解之间是否存在矛盾或遗漏？
2. 识别需求中的模糊点和假设——哪些功能描述不够具体？
3. 评估5个并行任务轨道与主线U+项目的集成点——各任务如何嵌入主线开发？
4. 给出最终的需求分析结论——这个实训项目到底要做什么？交付什么？

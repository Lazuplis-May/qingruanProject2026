# 审议式代码审查 — 用户指令原文

时间戳：202606301429
流程：deliberative-harness / code-review

## 指令原文

启动审议式代码审查流程，审查前端代码是否有功能性遗漏，页面缺失，修饰样式不生效的状况，依据详细设计文档docs/2_detailed_design_v4.md，界面交互原型docs/prototype.html。审查只需要关注代码与设计文档。你计划进行几轮评审，启动2个子Agent，每个Agent每次执行1轮审查，
并行进行，直到全部轮次完成，将所有reviews/此次流程/review_v*.md文件中的严重和一般问题整理到reviews/此次流程/todo.md，保留题目、位置、描述、忽略建议

## 关键参数

- 审查对象：前端代码（项目根 src/）
- 依据文档：docs/2_detailed_design_v4.md（详细设计）、docs/prototype.html（界面交互原型）
- 审查维度：功能性遗漏、页面缺失、修饰样式不生效
- 审查边界：只关注代码与设计文档对照
- 并行度：2 个子 Agent / 轮，每个 Agent 每次执行 1 轮审查
- 轮次：由主 Agent 决定（定为 2 轮，共 4 份 review_v*.md）
- 产出汇总：reviews/此次流程/todo.md，保留字段：题目、位置、描述、忽略建议
